前端如何用gulp工具来自动化自己的项目

发表于 2016-12-15   |   分类于 node   |     |   阅读次数

好久没有更新了，实在是太忙了，一个项目接着一个，最终还是因为懒。
之前写过一个gulp感觉写得不是很详细，也是入门的时候，有很多东西不理解，所以今天进一步的去写一下gulp吧

gulp的工作原理其实就是文档流，就是干的事情有先有后，就是这个工具处理完返回一个文档流给下一个工具再来处理，最后在转成文件的形式(个人理解)

这里主要说gulp的配置，用它来为我们的项目来服务，那么我们先来看看我的这个配置都能干嘛

功能

  • 编译less成css
  • 给css自动属性加前缀
  • 将px按照一个基数转化成rem
  • 合并css，合并js
  • 压缩css，压缩混淆js
  • 监听文件的变化

文件结构

  • app(js、css、index.html)
  • images
  • dist(编译之后自动)
  • gulpfile.js(配置文件)
  • package.json(包)

gulp配置

执行gulp会默认执行名字叫做 default的任务，其实写全了就是gulp default

后期改进

根据需求会不断的增加新的功能

  • 转es6
  • 提取页面出现的字的特殊字体 (字蛛)[https://github.com/aui/gulp-font-spider]，或者其他方式
  • 压缩图片(建议直接在网上在线转)
  • html自动写入css，js路径

css3-animation

不知道大家对它是不是很熟悉了呢
animation: name duration timing-function delay iteration-count direction
animation 属性是一个简写属性，用于设置动画属性：

  • animation-name:动画名称，需要用@keyframes去设置一个动画
  • animation-duration:动画完成的周期，0则不会播放动画
  • animation-timing-function:动画速度的曲线
  • animation-delay:延迟执行动画的时间,也可以负值
  • animation-iteration-count:动画被播放的次数
  • animation-direction:播放方向
  • animation-fill-mode:设置动画结束后的状态
